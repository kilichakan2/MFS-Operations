## Executive summary

This PR ships the fourth Phase 0 foundation: a single ESLint configuration edit that forbids
`import ... from '@supabase/supabase-js'` everywhere in the project except three explicitly
allow-listed locations ‚Äî the existing central client file (`lib/supabase.ts`), the prospective
Phase 1+ adapter directory (`lib/adapters/supabase/**/*.ts`, not yet created), and the test
infrastructure (`tests/**`). The forbidden-import rule runs at severity `"error"` so it
blocks merge once CI is in place, not merely warns. A Vitest unit suite at
`tests/unit/lint/no-supabase-sdk.test.ts` pins the configuration's behaviour through six cases
using ESLint's `ESLint` class (not `Linter`, which silently ignores `overrides[]`), running the
rule in-process against inline string fixtures ‚Äî no disk I/O, no shelling out. F-04 lands with
zero new app-code violations because both existing SDK importers (`lib/supabase.ts:13` and
`tests/integration/_setup.ts:24`) are allow-listed. This PR is the implementation of the FREEZE
rule stated verbatim in ADR-0003 lines 21-22.

### L4 ‚Äî TypeScript

Branch `npx tsc --noEmit 2>&1 | wc -l` returns **107**. Main returns **103**. The 4-line
discrepancy is a calibrated flag ‚Äî not a blocker ‚Äî explained as follows:

The `@types/eslint` devDependency (added by the Render-phase amendment) pulls in two transitive
type packages: `@types/estree` and `@types/json-schema`. The `@types/json-schema` package was
not present anywhere in main's dependency tree (confirmed: not in main's flat `node_modules/`
nor in any nested `node_modules/`). Its presence on the branch changes tsc's global type
resolution and surfaces four latent `TS2322`/`TS2345` errors in four pre-existing component
files:

- `components/BottomNav.tsx:54` ‚Äî `TS2322: Type 'string' is not assignable to type 'UrlObject | RouteImpl<string>'`
- `components/DesktopSidebar.tsx:101` ‚Äî same
- `components/MoreDrawer.tsx:85` ‚Äî same
- `components/PwaGuard.tsx:66` ‚Äî `TS2345: Argument of type 'string' is not assignable to parameter of type 'RouteImpl<string>'`

All four are in `components/` files that F-04 does not touch. All four are pre-existing
typing issues with Next.js `Link` component `href` props typed as `string` where Next.js's
typed routing expects `RouteImpl<string>`. The errors were always present but invisible on
main because `@types/json-schema` was absent. F-04 files (`tests/unit/lint/no-supabase-sdk.test.ts`
and `.eslintrc.json`) produce zero tsc errors ‚Äî confirmed by `npx tsc --noEmit 2>&1 | grep "tests/unit/lint/"` returning empty.

**Calibrated verdict for L4:** zero NEW errors in F-04 files (strict pass). The +4 surfaced
errors are latent pre-existing issues in pre-existing files; they are carried forward as an
advisory item (see ¬ßCarry-forward advisory items). The L4 layer is PASS on the calibrated

## Carry-forward advisory items

1. **`@types/json-schema` (transitive dep of `@types/eslint`) surfaces 4 latent tsc errors
   in pre-existing component files.** The four components (`BottomNav.tsx`, `DesktopSidebar.tsx`,
   `MoreDrawer.tsx`, `PwaGuard.tsx`) pass `href: string` to Next.js's `Link` component where
   Next.js typed routing now expects `RouteImpl<string>`. These errors existed silently before
   F-04 and would have been surfaced by any future dep that brings in `@types/json-schema`.
   F-04 is not the root cause; it is the first PR to expose them. The conductor should flag
   these four files as F-TD-01 items for the next tech-debt cleanup PR. No action in F-04.

2. **The FREEZE rule covers `import` statements only ‚Äî not CommonJS `require()`.** Documented
   in the plan's Risk 2. Advisory only; the codebase is ESM throughout.

3. **`tests/**`is broader than ADR-0003 strictly requires.** The override could be tightened
to`tests/integration/\_setup.ts` (the single current legitimate use site) without breaking
anything today. Recommend leaving it broad (`tests/\*\*`) per the plan's decision and tightening
   only if misuse appears in a future PR. Conductor to note at Gate 4.

4. **F-05 (Orders/Customers/Products port extraction) is now blocked from adding new SDK
   imports.** F-04 is in effect. Any F-05 work that tries to add `from '@supabase/supabase-js'`
   outside the allow-list will fail lint immediately. F-05's implementer should wire through
   `supabaseService` from `@/lib/supabase` or create a new adapter file under
   `lib/adapters/supabase/` which is pre-emptively allow-listed.

5. **CI enforcement.** F-04 lands with lint enforced locally only (via ANVIL Gate 3 and the
   calibrated `npm run lint` discipline). When CI is configured in a future unit, the FREEZE
   rule gains automatic enforcement on every PR. Until then, code review + ANVIL is the guard.
