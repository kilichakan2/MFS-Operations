## Executive summary

F-08 shipped the server-side duplicate-order guard; F-TD-10 switches it on from the
client. The new-order form now attaches a one-time random fingerprint (UUID) to each
"place order" request: a double-tap or wifi retry reuses the same fingerprint and the
server returns the original order instead of creating a duplicate; editing any order
detail (customer, date, notes, lines) forgets the fingerprint so a changed order is
always genuinely new; success forgets it so the next order starts fresh.

In plain English: one tap = at most one order, always; an edited order is always a new
order; nothing about the server changed.
