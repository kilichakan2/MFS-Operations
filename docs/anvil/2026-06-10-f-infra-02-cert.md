## Executive summary

F-INFRA-02 is pure test-and-deploy plumbing. It delivers the Gate-4 preview smoke: one command
(`npm run test:e2e:preview -- <preview-url>`) that runs the `@critical` Playwright paths against
a PR's Vercel preview deployment + Supabase preview branch, with defence-in-depth guards that
refuse production targets before any network traffic flows.

ANVIL's contribution this run: a new "guard the guards" unit test file —
`tests/unit/scripts/e2e-preview-guards.test.ts` (11 black-box tests, commit `973acb9`) — that
spawns the real wrapper script as a child process with hostile inputs (production hostname,
`-git-main-` alias, plain `http://`, foreign hostnames, the production Supabase project ref,
malformed URLs, missing/short bypass secret, no argument) and proves every refusal exits
non-zero, with the right message, before the Playwright launch line, near-instantly, and without
ever printing the bypass secret.

Layers 1–5 are all green. Notably, Layer 5's `@critical` run passed 8/8 from a fresh
`db:reset` with NO manual fixture step — proving the PR's central claim: the updated
`supabase/seed.sql` ANVIL-TEST block alone makes a virgin database (and therefore a virgin
Supabase preview branch) test-ready.

**Layer 6 — preview self-proof — PASSED on 2026-06-11** (Step 0 platform wiring completed +
`npm run test:e2e:preview` against PR #26's own preview deployment, 8/8 `@critical`). The remote
plumbing this PR introduces is now proven against a real preview target wired to a real Supabase
preview branch. See the "Layer 6 — preview self-proof" section for the full record, including
the fail-closed refusal that preceded it.

A revert of the PR's merge commit is the complete rollback:

```bash
git log --oneline main | head -3      # find the squash-merge SHA

git push origin main
```

`db:reset` runs and FUTURE Supabase preview branches — it cannot retroactively alter any
