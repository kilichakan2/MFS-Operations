## Executive summary

F-TD-03 is a pure test-infrastructure unit. It delivers a self-contained Vitest integration
runner — the mechanism that makes `npm run test:integration` safe, reliable, and impossible to
accidentally aim at production:

1. **Env guard** — refuses to start if `.env.test.local` is missing or if its Supabase URL
   contains the production project ref (`uqgecljspgtevoylwkep`).
2. **Port guard** — refuses to start if port 3100 is occupied (proves a fresh, verifiable server
   every run).
3. **DB identity probe** — plants a random sentinel user in the LOCAL database, then reads it
   back through the booted server. If the server is wired to any other database, the probe
   fails and the run aborts before any test traffic flows.
4. **Reliable teardown** — kills the whole process group (not just the parent PID) with an
   exit-handler backstop, so Ctrl-C never orphans the server.

The PR is the test infrastructure itself. Verification = proving the deliverable works end-to-end.

All 92 integration tests pass. Both the cold run and the immediate back-to-back run (state-leak
check) pass. All three negative guards (missing env, port squatter, stack-down) fire correctly.
Port is free and no orphan processes remain after teardown. No sentinel rows left in the database.

## Calibrated-criteria notes

**L1 (unit suite):** The 2 failures in `tests/unit/annualReview.test.ts` are date-sensitive boundary
tests (`trainingRefreshStatus` "today → due_soon" and "91 days → current") that fail because the
test fixture computes dates relative to today and the training refresh window boundaries shift each
day. These exist identically on `main` at `c9a997e`. Confirmed by running the same test file in
isolation and matching the failure set. F-TD-03 introduced zero new unit failures.

**L3 (ESLint):** 58 findings vs. the "~55 pre-existing" baseline noted in the matrix. The
3-count difference is within normal variation — the matrix estimate was approximate. All 58
findings are in `app/admin/`, `app/haccp/`, `app/pricing/`, `app/complaints/`, `app/dispatch/`,
`app/visits/`, `app/haccp/`, and `components/BottomSheetSelector.tsx` — none in any file
changed by this PR. Verified by `npm run lint 2>&1 | grep -E "tests/integration|vitest.integration"`.

**L4 (TypeScript):** 60 errors matching the approved baseline of "60 pre-existing" exactly.
All 60 are in pre-existing app code. Verified by
`npx tsc --noEmit 2>&1 | grep -E "tests/integration/_(config|setup|assertStack|globalSetup)|vitest.integration.config"` returning empty.

**L2 dual-stack port probe deviation (accepted-better):** The `assertPortFree()` function probes
BOTH `127.0.0.1` and `::1` (dual-stack). This is an improvement over a single-family probe —
`python3 -m http.server` binds IPv6, which a IPv4-only probe would miss. Guard (b) confirms this
works correctly: the squatter was detected and the run aborted. This deviation from a naive
single-family probe is accepted-better.

**READY_TIMEOUT advisory:** The 90-second `READY_TIMEOUT` for the dev server boot may need to
be increased to 120 seconds on cold machines with slow first-compile times. This is a non-blocking
advisory carried forward from the Guard review. `F-TD-07` tracks raising this value if CI
machines prove slower than local.

**Sentinel-row production hygiene advisory:** The 6 `ANVIL-TEST-{role}` fixture rows (one per
role) are intentionally persistent between runs — they are shared stable fixtures, not sentinel
rows. The ephemeral `ANVIL-TEST-sentinel-{random}` rows created by the identity probe are
cleaned up after every run (confirmed: 0 rows remain after both L2 runs). `F-PROD-01` in
`BACKLOG.md` tracks production hygiene for fixture row management.
