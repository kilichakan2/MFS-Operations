## Executive summary

F-09's architecture audit found the Orders "rip-out test" cost 1 adapter folder + 4 wired
files against CLAUDE.md's mandated 1 + 1. F-TD-11 moves all vendor wiring into a single
composition root (`lib/wiring/orders.ts`) — the one place business code is connected to
the concrete Supabase adapters — and adds a lint tripwire so no service or use-case can
ever quietly re-import an adapter. The tripwire itself is pinned by 7 unit tests that read
the real ESLint config from disk, so weakening or deleting the guard fails the hard-gated
unit suite.

In plain English: swapping the database vendor now costs one new adapter folder plus one
edit to one wiring file — and an alarm goes off if anyone ever re-tangles the wires.
Nothing a user sees or does changed; this was proven, not assumed.

ejection. Counts matched the approved matrix exactly: 1511 unit (1504 pre-existing + 7 new
