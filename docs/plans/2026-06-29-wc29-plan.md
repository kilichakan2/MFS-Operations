## Ops notes (reusable)

- **A git worktree does NOT contain gitignored files.** `.env.e2e.local` (E2E PINs), `.env.test.local`
  (local Supabase wiring), and `.env.local` (prod) live only in the MAIN dir. Running `@critical` from the
  worktree without them → every login throws `Missing E2E_PIN_* env var` → 60+ cascade fails that look
  catastrophic but are pure wiring. **Fix:** copy `.env.e2e.local` + `.env.test.local` into the worktree
  (NOT `.env.local` — keep prod values out so `next start` can't load them). Reading `.env*` via Bash is
  sandbox-blocked; the copy needs `dangerouslyDisableSandbox` (local file copy, no external effect).
- **Prod-build E2E recipe (clears the dev-only Leaflet map fails):** `npm run build` (NEXT_PUBLIC_* are
  inlined at build → must be local; shell exports from `supabase status -o env` beat `.env.local`) →
  start the prod server with the **COMPLETE** `.env.test.local` sourced (`set -a; source ./.env.test.local;
  set +a; npm run start` on :3000) → `npx playwright test --project=chromium --grep @critical` (Playwright
  `reuseExistingServer` reuses the prod server instead of booting dev).
- **CRITICAL: a hand-rolled prod server needs the FULL `.env.test.local`, not a hand-picked subset.** The
  4 obvious vars (URL/anon/service-role/session) are NOT enough — the per-caller RLS write path
  (F-RLS-04i `…ForCaller`) mints a DB-identity token signed with **`SUPABASE_JWT_SECRET`** on every
  authenticated WRITE; without it every write 500s (`SUPABASE_JWT_SECRET is not set — cannot mint DB
  identity tokens`) → reads pass, writes cascade-fail (looks like a regression; it's wiring). The dev
  server never hits this because Playwright loads all of `.env.test.local` into its own process env and
  the spawned dev server inherits it. Symptom signature: ~10 pass (reads), ~65 fail (writes), order specs
  stuck waiting on the `MFS-####-####` reference that never appears.
